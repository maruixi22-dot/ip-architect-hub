# IP Architect Hub - 系统架构设计

## 1. 系统概述

IP Architect Hub 是一个**B2B SaaS 平台**，为自媒体从业者和机构提供 IP 孵化全流程解决方案。

### 1.1 核心价值

- **效率提升**：自动化内容采集、改写、发布流程
- **数据驱动**：实时监控多平台数据，科学决策
- **规模化管理**：统一管理多个 IP 项目
- **AI 赋能**：智能化运营规划和内容生成

### 1.2 目标用户

- 自媒体从业者（美妆、科技、美食等领域）
- MCN 机构
- 品牌方市场团队
- 个人创业者

## 3. 核心模块设计

### 3.1 IP 调研模块

**功能**：通过问卷和语音收集 IP 信息，AI 生成人物画像

**流程**：
```
用户输入 → 语音识别 → 文字处理 → AI 分析 → 画像生成 → 存储
```

**接口设计**：
```
POST /api/research/create     # 创建调研
POST /api/research/voice      # 上传语音
POST /api/research/submit     # 提交问卷
GET  /api/research/:id        # 获取调研结果
POST /api/research/analyze    # AI 分析
```

**数据模型**：
```typescript
interface IPResearch {
  id: string
  userId: string
  projectId: string
  answers: ResearchAnswer[]
  audioUrl?: string
  transcript?: string
  persona: IPPersona
  status: 'draft' | 'completed' | 'analyzing'
  createdAt: Date
}

interface IPPersona {
  name: string
  category: string // 美妆/科技/美食/等
  positioning: string
  targetAudience: string
  contentStyle: string
  uniqueValue: string
  tags: string[]
}
```

---

### 3.2 运营规划生成模块

**功能**：基于调研数据，AI 自动生成运营规划书

**流程**：
```
调研数据 → AI 生成 → 模板渲染 → 人工审核 → 发布
```

**接口设计**：
```
POST /api/plan/generate        # 生成规划
GET  /api/plan/:id            # 获取规划
POST /api/plan/:id/approve    # 审核通过
POST /api/plan/:id/export     # 导出 PDF
```

**数据模型**：
```typescript
interface IPPlan {
  id: string
  projectId: string
  researchId: string
  content: {
    positioning: string
    targetAudience: string
    contentStrategy: ContentStrategy[]
    publishingSchedule: Schedule
    growthTargets: GrowthTarget[]
    riskAssessment: string
  }
  status: 'draft' | 'reviewing' | 'approved'
  createdAt: Date
}
```

---

### 3.3 内容采集模块

**功能**：定时采集小红书/抖音热门内容

**流程**：
```
定时任务 → 爬虫采集 → 内容解析 → AI 筛选 → 存储入库
```

**接口设计**：
```
POST /api/collector/start      # 启动采集
GET  /api/collector/status     # 采集状态
GET  /api/collector/contents   # 内容列表
POST /api/collector/filter     # AI 筛选
```

**爬虫策略**：
- **小红书**：爬取热门笔记、关键词搜索
- **抖音**：爬取热门视频、话题挑战
- **频率控制**：避免被反爬
- **去重机制**：Redis 布隆过滤器

---

### 3.4 推送系统

**功能**：定时推送优质内容到用户

**流程**：
```
定时触发 → 筛选内容 → 渲染模板 → 多渠道推送 → 记录日志
```

**推送渠道**：
- 🦞 **飞书**（已配置）
- 📧 **邮件**
- 🌐 **网站通知**

**接口设计**：
```
POST /api/push/schedule       # 设置推送时间
GET  /api/push/history        # 推送历史
POST /api/push/test           # 测试推送
```

---

### 3.5 AI 改写模块

**功能**：一键改写内容，符合 IP 调性

**流程**：
```
选择内容 → 改写配置 → AI 改写 → 预览 → 确认
```

**改写模式**：
- 保持原意
- 风格转换（活泼/专业/幽默）
- 长度调整（长文/短文）
- 平台适配（小红书/抖音）

**接口设计**：
```
POST /api/rewrite             # 开始改写
GET  /api/rewrite/:id         # 获取结果
POST /api/rewrite/batch       # 批量改写
```

---

**功能**：自动发布到多个平台

**流程**：
```
内容审核 → 格式化 → 定时发布 → 状态追踪 → 数据回传
```

**支持平台**：
- 📕 小红书
- 🎵 抖音
- 📱 快手
- 🐦 微博

**接口设计**：
```
POST /api/publish/platform      # 绑定平台
GET  /api/publish/accounts     # 账号列表
POST /api/publish/content      # 发布内容
GET  /api/publish/status/:id   # 发布状态
```

---

### 3.7 数据分析模块

**功能**：实时监控 IP 数据表现

**监控指标**：

- 内容互动数据（点赞/评论/转发）
- 内容爆款分析
- 竞品对比

**可视化图表**：
- 折线图（趋势）
- 柱状图（对比）
- 饼图（占比）
- 热力图（时段）

**接口设计**：
```
GET /api/analytics/overview    # 总览数据
GET /api/analytics/trends      # 趋势数据
GET /api/analytics/competitor  # 竞品对比
GET /api/analytics/report      # 生成报告
```

**功能特性**：

- 项目配置

## 4. 数据流设计

### 4.1 用户操作流程

```
注册/登录 → 创建IP项目 → IP调研 → 生成规划
    ↓
设置采集规则 → 系统定时采集 → 内容推送
    ↓
用户挑选内容 → AI改写 → 多平台发布
    ↓
数据采集 → 分析报告 → 优化策略
```

### 4.2 数据采集流程

```
定时任务触发 → 调用爬虫 → 解析内容
    ↓
AI 评分筛选 → 存入内容库
    ↓
推送任务 → 用户接收 → 用户挑选
    ↓
改写任务 → 发布任务 → 数据回传
```

## 5. 安全设计

### 5.1 身份认证

- **JWT Token**：无状态认证
- **OAuth 2.0**：第三方登录
- **Session**：Redis 存储

### 5.2 权限控制

- **RBAC**：基于角色的权限控制
- **数据隔离**：多租户数据隔离
- **API 限流**：防止滥用

### 5.3 数据安全

- **加密存储**：敏感数据加密
- **HTTPS**：传输加密
- **SQL 注入防护**：参数化查询
- **XSS 防护**：输入过滤
